核心概念：将“宿舍关系”游戏化 (Gamification)
不要把它仅仅做成一个“查询工具”，而要把它做成一个**“宿舍管理控制台” (Dorm Dashboard)**。风格可以采用 赛博朋克 (Cyberpunk) 或 像素风 (Pixel Art)，把宿舍生活比喻成一场 RPG 游戏。

应用定位
“基于多维性格分析与博弈论的宿舍协作辅助系统” (听起来是不是立刻高大上了？)

🚀 功能模块设计 (由浅入深)
1. 舍友雷达系统 (The Roommate Radar)
痛点： MBTI 只有 4 个维度，不够精准。

功能： 用户输入舍友的 MBTI 或回答几个关键问题（如：闹钟响度偏好、空调温度偏好、卫生洁癖度、熬夜指数）。

技术亮点 (Canvas/SVG Visualization)：

不只是显示 "INFP"，而是生成一个动态的 六维雷达图 (Radar Chart)。

算法实现： 编写一个前端算法，计算你与舍友的**“重合面积”和“冲突极点”**。

例子：你的“睡眠敏感度”高，舍友的“键盘敲击声”大，系统会在雷达图上标记出红色警告区域。

2. “非暴力沟通”翻译器 (NVC Translator)
痛点： 不好意思直接开口提意见（比如“你脚好臭”、“别半夜打游戏”）。

功能： 用户输入一句充满了情绪的“脏话”或抱怨，系统将其“翻译”成委婉、得体、符合非暴力沟通 (Non-violent Communication) 原则的建议文案。

技术亮点 (NLP/Rule-based Logic)：

初级版： 使用关键词匹配 + 模板替换（例如检测到“吵”，随机生成三种委婉话术）。

进阶版（比赛加分项）： 接入轻量级的 OpenAI API (需用户填Key) 或使用浏览器端的微型模型 (如 TensorFlow.js) 进行简单的文本风格转换。

甚至可以生成一个二维码，让你匿名贴在宿舍群里。

3. 熵增对抗机制：家务扭蛋机 (Chore Gacha)
痛点： 没人倒垃圾，分工表枯燥无效。

功能： 把做家务变成抽卡游戏。

技术亮点 (Randomness & Animation)：

利用 CSS3 动画 或 Pixi.js 制作一个精美的“扭蛋机”。

输入宿舍所有人的名字，点击“抽卡”。

权重算法： 如果某人上周做了家务（记录在 LocalStorage），他这周被抽中的概率降低 50%。这展示了你对“加权随机算法”的理解。

4. 噪音分贝可视化监控 (Web Audio API)
痛点： “我现在说话大声吗？”

功能： 一个实时的仪表盘，显示当前环境噪音。

技术亮点 (Web APIs)：

调用浏览器的 Web Audio API 获取麦克风权限。

实时分析音频频谱，并用 D3.js 或 Three.js 渲染成酷炫的声波纹。

设置阈值：当分贝超过 60dB 时，页面背景变红并弹出警告：“警告：您已进入‘室友仇恨’区域”。

这能极大地体现前端对原生 API 的掌控能力。

🛠️ 技术栈推荐 (针对 GitHub Pages 优化)
为了在比赛中展示技术实力，建议放弃传统的 HTML/jQuery，采用现代工程化方案：

框架：React 或 Vue 3 (配合 TypeScript，体现类型安全意识)。

构建工具：Vite (速度快，配置简单)。

UI 库：

Tailwind CSS (原子化 CSS，手写样式显得专业)。

Framer Motion (React) 或 GSAP：制作丝滑的交互动画（比赛很看重“卖相”）。

数据存储：

Zustand / Pinia (状态管理)。

Redux-persist 或手动封装 LocalStorage：实现“刷新页面数据不丢失”，模拟后端体验。

可视化： ECharts (雷达图) 或 Three.js (如果想做一个 3D 宿舍模型作为主页背景)。

🏆 比赛制胜策略 (Pitch Deck)
在比赛演示时，你可以这样包装你的项目：

解决真问题： "大学生宿舍矛盾是全球性难题，我们用技术手段降低沟通成本。"

去后端化设计 (Serverless/Local-first)： "考虑到隐私（大家不想把宿舍八卦传到服务器），我们在 GitHub Pages 实现了纯前端的数据处理和逻辑闭环，数据只存在用户浏览器中，安全隐私。" —— 这是一个非常好的技术切入点。

算法可视化： 展示你的“性格重合度计算公式”或“家务权重算法”。

🖼️ 界面想象 (UI Mockup)
主页： 一个 2.5D 的像素风宿舍房间。点击床铺弹出“性格分析”，点击垃圾桶弹出“家务扭蛋”，点击电脑弹出“沟通翻译器”。

配色： 霓虹紫 + 荧光绿（赛博朋克），或者 柔和莫兰迪色（疗愈系）。
